배운 것 :Web
                Web Service : 웹 서비스(web service)는 네트워크 상에서 서로 다른 종류의 컴퓨터들 간에 상호작용을 하기 위한 소프트웨어 시스템이다.
                Terminal : 단말기 또는 터미널이라고 불린다. 데이터 통신시에 신호를 보내거나 받는 측의 한쪽 끝에 달린 장치를 말한다. 어떤 사람이 신호를 보낼 수 있는 장치를 가지고 있다면 그것
이 바로 단말기가 되는 것이다.
                        -In Linux : 컴퓨터와 사용자간의 소통시켜주는 인터페이스이다. 인터페이스란 텍스트 기반과 그래픽 기반(GUI) 두가지의 기반을 포함하는 용어이지만 리눅스 터미널은 텍스트 >기반을 말한다. 터미널은 shell을 기본적으로 사용하고 있으며 shell은 문자기반의 명령어들을 컴퓨터 언어로 변환하여 컴퓨터와 소통을 가능하게 해준다. 종류는 bash, tcsh, cmd(window)가 있다.
                HTTP : Hyper Text Transfer Protocol의 약자로 www상에서 정보를 주고받을 수 있는 프로토콜이다. client와 server사이에 이루어지는 요청과 응답 프로토콜이다. http를 통해 전달되는 자료는 http:로 시작하는 URL(인터넷 주소)로 조회할 수 있다.
                HTML : HyperText Markup Language의 약자로 웹페이지를 위한 지배적인 마크업 언어이다. 이것이 점점 발전하여 css나 jqery와 같은 마크업 언어가 표시되는 방법을 기술하는 언어들도 >개발되었다.


기억에 남는 것 : 교수님께서 web service가 뭐냐고 갑자기 물어보셨다.
                 찾아보려고 노트북을 가지고 왔는데 당황해서 인터넷이 아니냐고
                 대답했다. 아우 부끄러워;; 그리고 교수님께서는 대학 다닐 동안에
                 잡지를 꾸준히 읽어서 잡기에 능하셨다고 했다. 컴퓨터 전공자로
                 모르는 용어나 단어가 나올 때마다 찾는 것을 권하셨다.
